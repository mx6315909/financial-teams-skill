### ❌ 错误：Embedding 配置导致 Gateway 崩溃（2026-03-08）

**事件**：擅自修改 embedding 配置，添加不支持的 Bailian provider 导致 Gateway 崩溃失联

**错误原因**：
1. **配置位置错误** - 添加了顶级的 `"embeddings": { "default": "..." }`，正确位置是 `agents.defaults.memorySearch`
2. **不支持的 provider** - Bailian/text-embedding-v2 不在 OpenClaw 原生支持的 embedding provider 列表中
3. **没有先查文档** - 官方文档明确说明配置方式，没查就动手

**正确配置方式**：
```json5
agents: {
  defaults: {
    memorySearch: {
      provider: "gemini",  // 或 openai/voyage/mistral/local/ollama
      model: "gemini-embedding-001",
      remote: {
        apiKey: "xxx"
      }
    }
  }
}
```

**OpenClaw 支持的 embedding provider**：
- `openai` - OpenAI embeddings
- `gemini` - Gemini embeddings (gemini-embedding-001)
- `voyage` - Voyage AI embeddings
- `mistral` - Mistral embeddings
- `local` - 本地 node-llama-cpp 模型
- `ollama` - Ollama 本地服务

**Bailian 不支持** - 阿里云百炼的 embedding 模型不在支持列表中

**教训**：
1. 修改配置前**必须先查官方文档**
2. 不要擅自添加不支持的 provider
3. 配置位置要严格按照 schema
4. 重要修改前先备份配置

### ✅ 正确：浏览器配置修复（2026-03-11）

**问题**: OpenClaw browser 工具无法检测到 Playwright 安装的 Chromium

**原因**: 
1. `relayBindHost` 未设置，默认绑定 `127.0.0.1`（仅容器内可访问）
2. 浏览器进程在 Gateway 重启后不会自动恢复

**解决方案**:
```json
{
  "browser": {
    "relayBindHost": "0.0.0.0",  // 允许宿主机访问
    "executablePath": "/home/node/.cache/ms-playwright/chromium-1208/chrome-linux64/chrome",
    "headless": true,
    "noSandbox": true
  }
}
```

**修复步骤**:
1. 修改 openclaw.json 添加 browser 配置
2. 重启 Gateway
3. 手动启动浏览器：`browser action=start profile=openclaw`

**教训**: 
- 浏览器配置需要指定 `relayBindHost: "0.0.0.0"`
- Gateway 重启后需要手动启动浏览器进程
- 小红书等平台有 IP 风控，容器 IP 会被识别，建议用 Chrome 扩展 Relay

### 核心原则
1. **止损比止盈重要 100 倍** - 亏损 50% 需要涨 100% 才能回本
2. **单只股票≤20% 仓位** - 分散风险，避免个股暴雷
3. **大盘环境决定仓位** - 牛市 70-90%，震荡市 40-60%，熊市 0-20%
4. **右侧交易为主** - 突破/回调/底部三大买入信号
5. **指标组合使用** - MA 看趋势，MACD 确认强度，KDJ/RSI 找买卖点
6. **不懂不投** - 不碰不懂的股票，不赚超出认知的钱

### 关键知识点
- **量在价先** - 成交量变化往往领先于价格变化
- **背离是最有价值的信号** - 顶背离见顶，底背离见底
- **ROE 是核心财务指标** - 连续 5 年 ROE>15% 是好公司
- **金字塔建仓** - 价格越低买越多，摊薄成本
- **交易计划必须写下来** - 买入理由、仓位、止损位、目标位

### 心态控制
- 贪婪：设定目标收益率，达到后分批止盈
- 恐惧：买入前设定止损位，到了就执行（不恐慌）
- 侥幸：到达止损位坚决执行，不抱幻想
- 后悔：接受不完美，赚该赚的钱，错过不可惜
- 过度自信：连续盈利后反而降低仓位（冷静）

### 仓位管理核心要点（2026-03-11 更新）
**五大原则**:
1. **分散投资**: 单只≤20%，行业≤30%，5-10只股票
2. **动态调整**: 牛市70-90%，震荡市40-60%，熊市0-20%
3. **分批建仓**: 首次30-40%试探，确认后加仓
4. **盈利加仓，亏损减仓**: 绝不"越跌越买"
5. **永远保留现金**: 10-20%，机会来临时有"子弹"

**金字塔建仓法**:
- 价格越低买越多，摊薄成本
- 底部重仓，反弹容易回本
- 适用：基本面确认的好公司、长期下跌后的底部

**动态仓位调整**:
- 盈利10% → 可加仓至上限
- 亏损5% → 减仓50%
- 亏损10% → 清仓止损

### 交易日志与复盘体系（2026-03-11 更新）
**交易日志三大作用**: 记录 → 分析 → 改进

**5个核心复盘问题**:
1. 为什么买？（买入理由是否充分）
2. 为什么卖？（卖出理由是否合理）
3. 时机对吗？（买卖点是否最优）
4. 执行如何？（是否按计划执行）
5. 学到了什么？（可复用的经验）

**关键指标**:
- 胜率 = 盈利次数 ÷ 总次数（目标>50%短线，>40%长线）
- 盈亏比 = 平均盈利 ÷ 平均亏损（目标>2:1）
- 期望值 = (胜率×平均盈利) - (败率×平均亏损)（目标>0）
- 最大回撤 < 20%

**三级复盘体系**:
- **日复盘**: 每天收盘后15-30分钟
- **周复盘**: 每周日晚，统计胜率、盈亏比
- **月复盘**: 每月末，评估策略有效性
