# Error Tiers - 错误分级系统

基于 Hazel_OC 的经验，不再压制所有错误。

## 🟢 Cosmetic（可忽略）
**特征**:
- 瞬态 API 失败（自动重试成功）
- 轻微超时
- 非关键数据缺失

**处理**: 记录但不报告
**示例**:
- Tavily API 429（已重试）
- 网页抓取超时（备用源可用）

---

## 🟡 Operational（操作级）
**特征**:
- 可能影响任务完成
- 需要监控但不一定立即处理
- 24小时内可能自愈

**处理**: 
- 内部记录
- 24小时后未解决则升级报告
- 每日摘要中提及

**示例**:
- 邮件 API 持续失败
- 备份部分完成
- 技能更新检查失败

---

## 🔴 Critical（关键级）
**特征**:
- 影响核心功能
- 数据丢失风险
- 安全风险

**处理**: 
- 立即报告
- 停止相关任务
- 等待人类指示

**示例**:
- 数据写入失败
- API key 失效
- 安全警报
- 磁盘空间不足

---

## 压制预算规则

1. **最大压制时间**: 24小时
2. **强制升级**: 任何 operational 错误超过24小时必须报告
3. **数据丢失**: 永不压制，立即报告
4. **每日摘要**: 包含所有压制的 operational 错误统计

## 错误报告模板

```
🚨 错误报告 [级别]
━━━━━━━━━━━━━━━━━━
类型: [API/文件/网络/其他]
首次发生: [时间]
尝试次数: [N次]
影响: [描述]
建议: [修复方案]
```
