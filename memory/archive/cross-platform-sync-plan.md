# 跨平台记忆共享系统设计方案

## 🎯 目标
打通 Telegram ↔ WhatsApp ↔ 控制台，实现统一身份认知和记忆同步

### 用户身份映射

| 控制台 | N/A | 本地访问 |

**关键发现**: 三个平台都是同一个老大！需要建立统一身份。

## 🔧 技术方案

### 方案1: 统一用户标识文件（推荐）
创建 `USER-IDENTITY.md` 作为跨平台身份锚点：

```markdown
# 统一用户身份

## 核心标识
- **统一ID**: `user_xiaolin_001`
- **Telegram**: 8381915800
- **WhatsApp**: +8613963767577 (主), +8615205379618 (备)

## 跨平台规则
1. 任何平台收到消息，先检查是否匹配上述ID
2. 匹配则加载相同的 MEMORY.md 和 working-buffer.md
3. 任务状态在所有平台共享
```

### 方案2: 修改 Session 启动流程
在 `AGENTS.md` 中增加跨平台初始化：

```markdown
## 会话启动检查清单

### 第一步：识别用户身份
1. 读取 inbound_meta 中的 sender_id
2. 对照 USER-IDENTITY.md 查找统一身份
3. 如果匹配到已知用户，加载该用户的完整记忆

### 第二步：加载共享记忆
- [ ] 读取 MEMORY.md（长期记忆）
- [ ] 读取 memory/YYYY-MM-DD.md（今日+昨日）
- [ ] 读取 working-buffer.md（危险区域/进行中的任务）
- [ ] 读取 SESSION-STATE.md（会话状态）

### 第三步：确认任务连续性
- "继续上次的任务：[任务描述]"
- 或 "检测到新对话，正在同步跨平台记忆..."
```

### 方案3: 实时同步机制
使用文件系统作为共享存储（所有平台共用同一 workspace）：

```
workspace/
├── MEMORY.md              # 长期记忆（所有平台共享）
├── memory/
│   └── 2026-03-06.md     # 每日日志（所有平台共享）
├── working-buffer.md      # 进行中的任务（所有平台共享）
├── SESSION-STATE.md       # 会话状态（所有平台共享）
└── .config/
    └── user-identity.json # 用户身份映射
```

## ⚠️ 注意事项

### 隐私保护
- 绝不泄露老大的身份信息
- 跨平台同步仅限于同一用户的不同平台
- 不向任何第三方暴露平台关联关系

### 技术限制
- 目前无法做到真正的"实时"同步（依赖轮询或心跳）
- 冲突解决需要人工介入
- 控制台没有明确的"用户ID"概念
